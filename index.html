Jekyll - What is it?

Jekyll transforms your plain text into static websites and blogs. Jekyll removes the need of setting up databases. It allows the user to use reusable templates, such as a blog template, and write content in markdown. This means making site changes can happen in one place instead of potentially hundreds!

Setting up Jekyll

This section will walk you through how to get a basic Jekyll site running.

  1. Install Ruby
  2. In your terminal, run gem install jekyll bundler
  3. Create a directory for your site. In the terminal, use mkdir YOUR-SITE
  4. Create an index.html file with the basics.
  5. In your terminal, run jekyll serve
  6. Go to http://localhost:4000

The basic site has been created!

The Power of Jekyll

Jekyll allows the user to create reusable template pages. That means you don't have to repeatedly copy and paste code to create different pages for your site!

Let's get a broad overview of how Jekyll works. We'll be looking at how three parts work together.

  • Layouts
  • Includes
  • Variables

Take a look at the following code:


  <!DOCTYPE html>
    <html>
      <head>
        <meta charset="utf-8">
        <title>
          {{ page.title }}
        </title>
      </head>

      <body>
        {% include navigation.html %}
        {{ content }}
      </body>
    </html>

This entire thing is called a layout. Layouts can be reused. What makes it reusable?

Notice the curly braces. That is called liquid which allows Jekyll to insert things at those locations. The page.title is an example of a variable set from this page. The include navigation.html takes a separate html file and inserts it on this page. In this case, it takes the code for the navigation menu and puts it on this page.

Creating layout pages

Layout pages are reusable templates. You can have pages be built based off these layout pages. Here are some examples taken from jeremenichelli.io.


  <!DOCTYPE html>
  <html lang="en">

    {% include head.html %}

    {% include header.html %}

    <body>
      <section class="page-wrapper">
        {{content}}
      </section>
    </body>

    {% include footer.html %}
  </html>

In the above code, you can see the liquid again. Notice how the layout has {% include ... %}. This brings in smaller files of html. This helps keep your code cleaner! Now, check out another example.


  <!DOCTYPE html>
  <html lang="en">

    {% include head.html %}

    {% include header.html %}

    <body>
      <section class="post-wrapper">
        <p class="post-date">{{ page.date }}</p>
      </section>
      <h1 class="post-title">
          Building a blog using Jekyll. Creating custom&nbsp;layouts</h1>
      <article class="post-content">
        {{content}}
      </article>
    </body>

    {% include footer.html %}
  </html>

The above layout is for making blog posts. Notice how it automatically places the following pieces in the page:

  • head.html
  • header.html
  • page.date
  • content
  • footer.html

The one part that could be automated is the post-title. You wouldn't want that to be the post title for every blog post.

Creating includes

The following information explains includes. It is taken from the convert a site to Jekyll documentation.

Let’s suppose your default.html file is massive and hard to work with. You can break up your layout by putting some of the HTML code in include files.

Add a folder called _includes in your root directory. In that folder, add a file there called sidebar.html.

Remove your sidebar code from your default.html layout and insert it into the sidebar.html file.

Where the sidebar code previously existed in default.html, pull in your “include” like this:

{% include sidebar.html %}

You can break up other elements of your theme like this, such as your header or footer. Then you can apply these common elements to other layout files. This way you won’t have duplicate code.

Creating and using data

Jekyll has a great way to access lots of data on your site. You can save it in the _data folder and access it using liquid. Below is the documentation from the Jekyll website for how to use it.

The Data Folder

The _data folder is where you can store additional data for Jekyll to use when generating your site. These files must be YAML, JSON, or CSV files (using either the .yml, .yaml, .json or .csv extension), and they will be accessible via site.data.

Example: List of members

Here is a basic example of using Data Files to avoid copy-pasting large chunks of code in your Jekyll templates:

In _data/members.yml:

  
    -name:Eric Mill
    github: konklone

    -name:Parker Moore
    github:parkr

    -name: Liu Fengyun
    github: liufengyun

Or _data/members.csv:

  
  name,github
  Eric Mill,konklone
  Parker Moore,parkr
  Liu Fengyun,liufengyun

This data can be accessed via site.data.members (notice that the filename determines the variable name).

You can now render the list of members in a template:

  
    <ul>{% for member in site.data.members %}
        <li>
            <a href="https://github.com/{{ member.github }}">
                {{ member.name }}
            </a>
        </li>
      {% endfor %}
    </ul>
